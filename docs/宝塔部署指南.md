# 宝塔面板部署 USDT 支付中间件

本文档说明如何在云服务器上通过宝塔面板部署本项目（Node.js + SQLite + Redis + PM2 + Nginx）。

---

## 一、服务器与宝塔准备

### 1.1 云服务器

- 系统：**CentOS 7+** 或 **Ubuntu 20.04+**（推荐）
- 配置：1 核 2G 起步，生产建议 2 核 4G
- 安全组：放行 **80、443、8888（宝塔）、3000（可选，仅调试用）**

### 1.2 安装宝塔

未安装时，SSH 执行（以 CentOS 为例）：

```bash
yum install -y wget && wget -O install.sh https://download.bt.cn/install/install_6.0.sh && sh install.sh
```

安装完成后记下 **面板地址、用户名、密码**，浏览器登录。

### 1.3 宝塔内安装软件

在 **软件商店** 安装：

| 软件 | 说明 |
|------|------|
| **Nginx** | 反向代理、静态资源 |
| **PM2 管理器** | 或单独装 Node 后全局装 PM2 |
| **Redis** | 队列后端，必装 |

若用「Node 版本管理」：

- 安装 **Node 版本管理器**（软件商店）
- 添加 Node.js **18.x 或 20.x**，并设为默认

若没有 Node 版本管理：

- 可用 **PM2 管理器** 自带的 Node，或在 SSH 里用 nvm/二进制 安装 Node 18+。

## 二、安装 Node.js 与 Redis（未装时）

### 2.1 用宝塔安装 Redis

1. 软件商店 → 搜索 **Redis** → 安装
2. 安装完成后点击 **设置** → 可设密码（建议生产环境设置）
3. 默认端口 **6379**，本机连接地址：`127.0.0.1:6379`

### 2.2 确认 Node.js 版本

SSH 登录服务器后执行：

```bash
node -v   # 需 v18 或以上
npm -v
```

若未安装或版本过低，可用 Node 官方源（示例 Ubuntu）：

```bash
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs
```

### 2.3 安装 PM2（全局）

```bash
npm install -g pm2
```

---

## 三、上传与部署项目

### 3.1 上传代码

任选一种方式：

**方式 A：Git（推荐）**

```bash
cd /www/wwwroot
sudo git clone <你的仓库地址> usdt-payment
cd usdt-payment
```

**方式 B：本地上传**

- 本地打包项目（可排除 `node_modules`、`frontend/node_modules`、`.git`）
- 宝塔 **文件** → 上传到例如 `/www/wwwroot/usdt-payment`
- SSH 进入该目录：`cd /www/wwwroot/usdt-payment`

### 3.2 安装依赖

```bash
cd /www/wwwroot/usdt-payment

# 后端
npm install --production

# 前端
cd frontend
npm install
npm run build
cd ..
```

### 3.3 环境变量

```bash
cp .env.example .env
nano .env   # 或用宝塔文件管理器编辑
```

**必须修改项：**

| 变量 | 说明 | 示例 |
|------|------|------|
| `PORT` | 本机监听端口 | `3000` |
| `AES_SECRET_KEY` | 32 字符密钥 | 随机字符串 |
| `AES_IV` | 16 字符 IV | 随机字符串 |
| `JWT_SECRET` | JWT 密钥 | 随机字符串 |
| `REDIS_HOST` | Redis 地址 | `127.0.0.1` |
| `REDIS_PORT` | Redis 端口 | `6379` |
| `REDIS_PASSWORD` | Redis 密码（未设留空） | 空或密码 |
| `ADMIN_USERNAME` | 管理后台账号 | 自设 |
| `ADMIN_PASSWORD` | 管理后台密码 | 强密码 |

**可选：** `TRON_API_KEY`（TronGrid）、RPC 地址等，见 `.env.example`。

### 3.4 初始化数据库

```bash
cd /www/wwwroot/usdt-payment
npm run init-db
```

成功后会在 `data/` 下生成 SQLite 数据库文件。

### 3.5 创建日志目录（PM2 用）

```bash
mkdir -p /www/wwwroot/usdt-payment/logs
```

## 四、进程守护（PM2）

**进程守护**：用 PM2 保活三个 Node 进程，崩溃自动重启，服务器重启后也可自动拉起来。

### 4.1 三个进程说明

| 进程名 | 启动文件 | 作用 |
|--------|----------|------|
| **usdt-api** | `src/app.js` | HTTP 服务：管理后台 API、商户开放 API、静态后台页 |
| **usdt-task** | `src/task-worker.js` | 消费 Redis 队列：提币签名上链、充币/提币回调重试 |
| **usdt-scheduler** | `src/scheduler.js` | 定时任务：充币扫描、自动归集、回调入队、节点检查 |

三个都要常驻运行，缺一不可。

### 4.2 一键启动（推荐）

```bash
cd /www/wwwroot/usdt-payment
pm2 start ecosystem.config.js
```

会按 `ecosystem.config.js` 同时启动上面三个进程。

### 4.3 常用命令

```bash
pm2 list              # 查看进程列表与状态
pm2 logs              # 所有进程日志（实时）
pm2 logs usdt-api     # 只看 API 日志
pm2 restart all       # 重启全部
pm2 restart usdt-api  # 只重启 API
pm2 stop all          # 停止全部
pm2 delete all        # 从 PM2 中移除（不删代码）
```

### 4.4 开机自启（必须做一次）

```bash
pm2 save
pm2 startup
```

执行 `pm2 startup` 后，终端会提示一行命令（如 `sudo env PATH=... pm2 startup systemd -u root --hp /root`），**复制并执行**，这样重启服务器后三个进程会自动起来。

### 4.5 用宝塔「PM2 管理器」时

若在宝塔里装了 **PM2 管理器**，可不用命令行，在面板里添加 3 个项目：

| 项目名 | 启动文件 | 运行目录 |
|--------|----------|----------|
| usdt-api | `src/app.js` | `/www/wwwroot/usdt-payment` |
| usdt-task | `src/task-worker.js` | `/www/wwwroot/usdt-payment` |
| usdt-scheduler | `src/scheduler.js` | `/www/wwwroot/usdt-payment` |

环境变量会读项目目录下的 `.env`（部分 PM2 插件支持）；若不生效，需在 PM2 管理器里为每个项目配置与 `.env` 一致的环境变量。

## 五、Nginx 反向代理（宝塔）

### 5.1 添加站点

1. 宝塔 **网站** → **添加站点**
2. 域名：如 `api.yourdomain.com` 或 `yourdomain.com`
3. 根目录可先填：`/www/wwwroot/usdt-payment/frontend/dist`（仅前端）
4. PHP 选「纯静态」或关闭

### 5.2 配置反向代理

在站点 **设置** → **反向代理** → **添加**：

- **代理名称**：`usdt-api`
- **目标 URL**：`http://127.0.0.1:3000`
- **发送域名**：`$host`

再在 **配置文件** 里合并下面规则（或按宝塔界面等价配置）：

```nginx
# 管理后台静态 + API 反向代理
location / {
    root /www/wwwroot/usdt-payment/frontend/dist;
    try_files $uri $uri/ /index.html;
}

# 管理后台 API（前端请求 /admin/api/...）
location /admin/ {
    proxy_pass http://127.0.0.1:3000;
    proxy_http_version 1.1;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}

# 开放 API（商户对接 /open/...）
location /open/ {
    proxy_pass http://127.0.0.1:3000;
    proxy_http_version 1.1;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}
```

说明：

- 访问 `https://yourdomain.com/` → 管理后台（Vue 打包后的静态）
- 访问 `https://yourdomain.com/admin/` → 转发到 Node 的 3000 端口（管理 API + 登录等）
- 访问 `https://yourdomain.com/open/` → 商户开放 API

若你希望 **API 用子域名**（如 `api.yourdomain.com`），可再建一个站点，只做反向代理到 `http://127.0.0.1:3000`，不配 `root`。

### 5.3 SSL（推荐）

在站点设置里 **SSL** → 申请 Let’s Encrypt 或上传证书，并开启 **强制 HTTPS**。

## 六、部署检查清单

- [ ] 宝塔已装：Nginx、Redis、Node 18+
- [ ] 项目目录：`/www/wwwroot/usdt-payment`（或自选）
- [ ] `npm install`、`frontend` 已 build
- [ ] `.env` 已配置（含 Redis、AES、JWT、管理员账号）
- [ ] `npm run init-db` 已执行
- [ ] PM2 已启动三个进程：`usdt-api`、`usdt-task`、`usdt-scheduler`
- [ ] PM2 已 `save` + `startup` 开机自启
- [ ] Nginx 反向代理已配置，管理后台与 API 可访问
- [ ] 防火墙/安全组放行 80、443

## 七、常用命令

```bash
# 进入项目
cd /www/wwwroot/usdt-payment

# 查看进程
pm2 list
pm2 logs usdt-api

# 重启
pm2 restart all

# 更新代码后
git pull
npm install --production
cd frontend && npm run build && cd ..
pm2 restart all
```

## 八、数据与备份

- **SQLite 库**：`data/payment.db`（以及其它 `data/*.db` 若有）
- **日志**：`logs/` 下 PM2 及应用日志
- 建议在宝塔 **计划任务** 中定期备份 `data/` 目录到其他磁盘或对象存储。

按上述步骤即可在宝塔环境下完成从本机到云服务器的迁移与部署。
